# OmegaWTK (Omega Widget ToolKit)
 A cross-platform UI written in c++

## Why??

We are living in an age where people demand simple straightforward API's without the hassle of recoding everything. Even with AI's help, tokens are still being burned and it's more important than to ever to conserve code length and focus on creating amazing applications and tools. There are issues with current cross-platform UI engines.
1. QT is expensive for professional use, it is old, and is slowly fading out of popularity. 
2. Chromium Embedded Framework: Electron/React Native uses the Chromium engine to render its ui which is web based language bridge. (I assume this would huge performance slowdown with this.) Chromium is a massive repo that was built during the browser wars and is bloated tons of extra features. (200MB+ install size). There are also limitations to the HTML spec, so customization of the application is limited by what the browser can do.
